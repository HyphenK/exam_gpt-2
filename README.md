## 1. Tokenizer
원본 GPT-2(그리고 이후의 모델들)와 가장 결정적인 차이를 가지는 부분으로, 원본 GPT-2가 단어 단위로 토큰을 생성하는 것과 비교하여, 본 코드는 알파벳과 문장부호 등의 글자 한 개 한 개에 대하여 토큰을 생성합니다.

예시로서,

```python
chars = sorted(list(set(text)))
stoi = {ch: i for i, ch in enumerate(chars)}
itos = {i: ch for ch, i in stoi.items()}
```

이 부분에서 전체 텍스트에 등장하는 문자들을 모아 vocabulary를 만들고, 각 문자를 숫자 ID로 바꾸는데,

```text
"hello"
```

를 대략

```text
['h', 'e', 'l', 'l', 'o']
```

처럼 문자 단위로 보고 토큰을 생성합니다.

## 2. Embedding

### 계산 가능한 형태로의 변환

위의 예시에서,

```text
['h', 'e', 'l', 'l', 'o']
```

는 토큰화를 알파벳 순으로 한다고 가정하였을 때(이는 코드 내에서 실제로 이루어지는 과정과 크게 다르지 않음), 대략 다음과 같이 이루어집니다.

```text
[8, 5, 12, 12, 15]
```

이러한 숫자만으로는 예측을 하기 위한 어떠한 연산을 하기 어렵기 때문에, one-hot encoding을 이용하여 전체 N개(코드에서는 64개)의 수를 갖는 리스트(벡터) 중 8의 경우 8번째 수만 1이고 나머지는 0, 5의 경우 5번째 수만 1이고 나머지는 0, ...의 방법으로 토큰을 변환합니다.

### 위치정보 추가

앞선 예시의 세 번째 l과 네 번째 l은 그 위치가 다르므로, 벡터에 주어지는 정보 또한 달라야 합니다. 이를 위해 앞선 Token Embedding 과정을 이용해 만든 리스트(벡터)에, 이 글자의 위치에 대한 정보도 추가한 것이 최종적으로 글자를 숫자 형태의 데이터로 변환한 결과가 됩니다.

## 3. Self-Attention

### 미래 정보에 대한 마스킹

Self-Attention은 각 토큰이 다른 토큰들을 참고해서 자신의 표현을 업데이트하는 구조인데, 이 때 예측하고자 하는 자신보다 뒤에 나오는 토큰을 미리 보는 것을 방지하기 위해, 계단 형태의 마스크를 씌워 각 토큰이 자신보다 앞쪽에 위치한 토큰만 참고할 수 있도록 강제합니다. 따라서

```text
1번째 토큰 → 1번째만 참고
2번째 토큰 → 1, 2번째만 참고
3번째 토큰 → 1, 2, 3번째만 참고
4번째 토큰 → 1, 2, 3, 4번째만 참고
```

형태가 되도록 합니다.

추가로, 학습 시에는 답을 미리 보는 행위를 막기 위해 이러한 구조가 사용되고, 예측 시에는 학습 시와는 다르게 미리 확인할 수 있는 답이 존재하지는 않으나, 각 토큰에 대한 계산 방법을 일치시키고자 마찬가지 구조의 마스크를 사용합니다.

### Query, Key, Value (Q, K, V)

Query(Q), Key(K), Value(V)는 Self-Attention 내에서 사용하는 변수로, 각각 자신이 찾고자 하는 것이 무엇인지, 자신이 어떤 특징을 가지는지, 자신이 실제로 어떤 정보를 가지고 있는지를 나타내는 값입니다. Q와 K를 이용하여, 자신과 가장 관련성이 높은 토큰이 어떤 것인지를 계산하는 Attention 점수를 구하고, 이 점수를 Softmax를 이용하여 합이 1인 확률분포로 변환한 후, 그 값과 V를 곱한 후 더하여 이 토큰의 정보가 자신과 관련성이 높은 토큰의 정보와 더 많이 연결되도록 합니다.

### 그리고 이것을 여러 개 사용

한 개의 Self-Attention은 한 종류의 관계에 대하여 계산해낼 수 있는데, 전체 입력되는 문장 내에는 여러 개의 관계가 있을 수 있으므로, 이러한 관계를 잘 찾기 위해 Self-Attention을 병렬로 여러 개 사용합니다.

## 4. 입력 토큰에 대한 다음 토큰 출력

이렇게 만든 토큰을 신경망에 입력하면 출력으로서, 다음 문자가 될 각각의 후보 토큰에 대한 점수가 나옵니다. 이 점수를 그냥 사용할 수는 없고 이것을 Softmax를 사용하여 합이 1인 확률 형태로 변환합니다. 그 확률분포에 의하여, 모델은 다음 토큰을 출력합니다. 이 때 기본 설정에서는 반드시 확률이 가장 높은 토큰이 출력되는 것은 아니고 (이 경우 지속적으로 같은 문장만 반복하여 출력하게 됩니다.) 확률분포에 의한 랜덤 선택 과정을 통해 출력할 토큰이 정해집니다.

## 5. 출력한 토큰이 정답인지 확인 후 결과를 피드백

예를 들어 원문이:

```text
hello
```

라면 학습 구조는:

```text
입력: h e l l
정답: e l l o
```

형태입니다. 즉, 매 위치에서 다음 문자를 예측하여 맞힐 수 있도록, 구체적으로는 입력에 대한 출력과 결과의 오차가 작아지도록 모델을 학습시킵니다. 오차 계산을 위한 Loss Function으로 Cross Entropy가 사용되며, 이렇게 계산된 오차를 이용해 모델의 가중치를 조절하는 Optimizer로는 Adam이 사용됩니다.

## 6. 잘 제작된 점

notebook_KJV.ipynb 파일의 최하단 출력부 출력을 통해, KJV 원문 특유의 고전 영문법 표현이 잘 등장하는 것을 확인할 수 있었습니다. 또한 고유명사에 대한 출력도 어느 정도 정확하게 가능하며, 지나치게 출력이 반복되거나 원문과 동일한 표현이 등장하는 등의 문제 또한 발견되지 않았습니다.

## 7. 개선 필요점

앞서 입력 토큰에 대하여 다음 토큰을 출력할 때에는 확률분포에 의한 랜덤 선택 과정을 통해 출력할 토큰이 정해진다 말한 바 있습니다. notebook_KJV.ipynb 파일의 최하단 출력부를 확인하면, 성경 특유의 1:1, 1:2, 1:3 등의 몇 장 몇 절을 표시하는 부분이 문장 앞에 표시되고 문장이 끝난 뒤 줄바꿈을 하고서, 다시 몇 장 몇 절을 표시하는 형태 자체는 모델이 잘 학습한 것을 확인할 수 있으나, 숫자의 순서는 모델이 학습하지 못 한 것을 확인할 수 있었습니다.

이를 해결하고자 notebook_KJV_v2.ipynb 모델에서는 temperature 값을 조절하여 출력 토큰을 결정할 때의 확률분포를, 확률이 높은 것을 조금 더 분명하게 확률이 높게, 그리고 확률이 낮은 것을 조금 더 분명하게 확률이 낮게 약간의 변형을 가한 후 출력 토큰을 결정하도록 변경해보았는데, 유의미한 변화가 관찰되지 않았습니다.

다음으로 notebook_KJV_v3.ipynb 파일에서는, 기존 파일이 최대 64개의 지난 토큰만 학습에 사용하는 것을 최대 128개의 지난 토큰을 학습에 사용하도록 변경하여 모델을 학습시킨 후 다시 결과물을 출력하도록 해보았는데, 마찬가지로 유의미한 변화가 관찰되지 않았습니다.

이는 현 코드의 Tokenizer가 글자 하나하나에 대하여 토큰화 과정을 수행하면서, 숫자에 대한 연속성 정보를 토큰에 담기 어려운 것이 원인인 것으로 추측됩니다. 따라서 이 문제를 해결하려면, 원본 GPT-2에서의 경우와 같이 글자 하나하나가 아닌 조금 더 큰 단위인 단어에 대하여 토큰화를 진행해야 유의미한 진전이 있을 것이라 생각됩니다.
